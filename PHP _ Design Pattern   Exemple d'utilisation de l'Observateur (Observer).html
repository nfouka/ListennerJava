Exemple d'utilisation de l'Observateur (Observer)

Publié le 10/10/2016.

Vous trouverez dans cet article un exemple d'utilisation du patron de conception Observateur (Observer) en PHP.

Définition

Ce patron de conception permet de limiter la dépendance entre les objets. Lorsque un événement précis se produit dans une application, il peut être nécessaire de lancer une ou plusieurs actions. Habituellement, toutes ces actions sont appelées à la suite dans une méthode. L'inconvénient de cette pratique est que la classe appelante devient dépendante de toutes les autres classes qui exécutent ces actions.

Grâce à ce patron de conception, on va pouvoir séparer les différentes actions. Pour cela, on crée un objet qui va être observé. C'est lui qui va lancé l'alerte. Ensuite, on va crée plusieurs objets qui vont observer cet objet. Dès que l'alerte est donnée par l'objet observé, les objets observateurs vont lancer leur(s) action(s).

Exemple

Notre exemple va consister à envoyer un e-mail aux membres et à mettre à jour le moteur de recherche lorsque l'on crée un article sur un blog. PHP définit des interfaces standards pour mettre en place l'Observateur. On va les utiliser dans notre exemple.

Notre première classe est simplement la classe métier Article. Pour simplifier l'exemple, elle n'a qu'un seul attribut, le titre de l'article :

class Article
{
    private $title;

    public function getTitle()
    {
        return $this->title;
    }

    public function setTitle($title)
    {
        $this->title = $title;

        return $this;
    }
}

Ensuite, on a la classe ArticleManager qui s'occupe de la gestion d'un article (création d'un article par exemple). C'est la classe qui va être observée. Elle va donc étendre la classe standard SplSubject. 3 méthodes doivent être implémentées. Une pour attacher des observateurs, une pour en détacher et la dernière pour les alerter :

class ArticleManager implements SplSubject
{
    /**
     * @var Article
     */
    private $article;

    /**
     * SplObjectStorage pour s'assurer de l'unicité des observateurs.
     *
     * @var SplObjectStorage
     */
    protected $observers;

    public function __construct()
    {
        $this->observers = new SplObjectStorage();
    }

    /**
     * @inheritdoc
     */
    public function attach(SplObserver $observer)
    {
        $this->observers->attach($observer);
    }

    /**
     * @inheritdoc
     */
    public function detach(SplObserver $observer)
    {
        $this->observers->detach($observer);
    }

    /**
     * @inheritdoc
     */
    public function notify()
    {
        foreach ($this->observers as $observer) {
            $observer->update($this);
        }
    }

    /**
     * Crée un article dans la base de données et alerte les observateurs.
     *
     * @param Article $article
     */
    public function create(Article $article)
    {
        $this->article = $article;

        // Ici dans un cas réel, la méthode va insérer l'article en base

        // Une fois tous les traitements effectués, on alerte les observateurs
        $this->notify();
    }

    /**
     * @return Article
     */
    public function getArticle()
    {
        return $this->article;
    }
}

La méthode create() permet de créer un article en base et d'alerter tous les observateurs.

On crée maintenant nos 2 classes observatrices : une qui envoie les e-mails, l'autre qui met à jour l'indexation du moteur de recherche. Elles doivent implémenter l'interface SplObserver. La méthode update() doit être implémenté. Pour simplifier au maximum, je donne uniquement le code devant être implémenté avec un affichage sur la console pour pouvoir tester :

class Notify implements SplObserver
{
    /**
     * @inheritdoc
     */
    public function update(SplSubject $subject)
    {
        echo("La classe Notify a été alerté. L'article '" . $subject->getArticle()->getTitle() . "' a été crée.\n");

        // Ici dans un cas réel, on envoie un e-mail à tous les utilisateurs avec un lien vers le nouvel article
    }
}

class SearchEngine implements SplObserver
{
    /**
     * @inheritdoc
     */
    public function update(SplSubject $subject)
    {
        echo("La classe SearchEngine a été alerté. L'article '" . $subject->getArticle()->getTitle() . "' a été crée.\n");

        // Ici dans un cas réel, on indexe le nouvel article
    }
}

On n'a plus qu'à tester le bon fonctionnement de l'ensemble des classes. On crée notre objet observé puis on y attache l'ensemble des objets qui vont l'observer :

$notify = new Notify();
$searchEngine = new SearchEngine();

$articleManager = new ArticleManager();
$articleManager->attach($notify);
$articleManager->attach($searchEngine);

Pou tester, on crée un article :

$article = new Article();
$article->setTitle('Titre de l\'article n°1');

$articleManager->create($article);

Les 2 objets vont alors bien être alertés :

La classe Notify a été alerté. L'article 'Titre de l'article n°1' a été crée.
La classe SearchEngine a été alerté. L'article 'Titre de l'article n°1' a été crée.
